# AGENTS.md

This file provides guidance to Codex (Codex.ai/code) when working with code in this repository.

## What This Is

A pnpm monorepo template for bootstrapping an AWS Organization with self-mutating CI/CD pipelines via CodePipeline, shared infrastructure (networking, security), and multi-app deployment. Clone this repo and follow the setup instructions to stand up a production-ready AWS Organization.

## Repo Layout

```
bin/            CDK entry point (app.ts)
config/         Three-layer config system (schema, defaults, local overrides)
lib/
  constructs/   Shared CDK constructs (Cognito, DynamoDB, SES, CloudFront CDN)
  infra/        Shared infrastructure stacks (networking hub, security, shared resources)
  pipeline/     Self-mutating pipeline stack definitions (infra + per-app)
  <app>/        CDK stacks for each app (created via org:add-app)
packages/
  <app>/        Runtime Lambda packages for each app
scripts/
  org/          AWS Organization management scripts (run once during setup)
  bootstrap/    CDK bootstrap helpers
test/           Node.js native test runner tests (compiled to dist/test/)
```

## Commands

```bash
# Install dependencies (also sets up Husky git hooks)
npm install

# Build TypeScript
npm run build                    # tsc -> dist/

# Run tests (builds first, then runs Node.js test runner)
npm test                         # npm run build && node --test dist/test/*.test.js

# Lint & format (Biome)
npm run check                    # lint + format check
npm run lint                     # lint only
npm run lint:fix                 # auto-fix lint
npm run format                   # apply formatting
npm run format:check             # check formatting

# CDK
npm run synth                    # synthesize CloudFormation
npm run deploy                   # deploy stacks

# Org management scripts (run in order during initial setup or account changes)
npm run org:build                # create OUs + accounts
npm run org:move                 # move accounts to target OUs
npm run org:bootstrap            # CDK bootstrap all accounts
npm run org:finalize             # generate config/defaults.ts from live AWS data
npm run org:setup-principal      # create OrgSetupRole + IAM Identity Center operator
npm run org:iam                  # create IAM Identity Center user + permission set
npm run org:scps                 # deploy Service Control Policies
npm run org:security             # enable CloudTrail, Config, GuardDuty, SecurityHub org-wide
```

## Config System (config/)

Three-layer config with typed schemas:

1. `config/schema.ts` -- TypeScript interfaces (`BootstrapConfig`, `AppConfig`, `DeploymentTarget`, etc.)
2. `config/defaults.ts` -- Generated by `npm run org:finalize` from live AWS data (checked in)
3. `config/local.ts` -- Optional gitignored overrides (copy from `local.example.ts`)

`config/load-config.ts` merges defaults + local overrides and validates (account IDs, CIDRs, connection ARNs, no overlapping CIDRs).

## CDK Entry Point (bin/app.ts)

Creates one `InfraPipelineStack` and one `AppPipelineStack` per app defined in config. When `CDK_BOOTSTRAP` env var is set, synthesizes a placeholder stack and exits (used by `org:bootstrap`).

## Shared Infrastructure (lib/infra/)

Stacks deployed to the Shared Services and Log Archive accounts via `InfraPipelineStack`:

- `shared-resources-stack.ts` -- Route 53 hosted zone and cross-account delegation role
- `network-hub-stack.ts` -- Transit Gateway egress VPC in Shared Services account
- `spoke-network-stack.ts` -- Spoke VPC per workload account
- `log-archive-stack.ts` -- S3 buckets for CloudTrail and Config logs
- `budget-alerts-stack.ts` -- AWS Budgets + SNS alerts

## Org Config (scripts/org/org-config.yaml)

Single source of truth for org structure, OU tree, account roles, IAM Identity Center settings, and pipeline config. All `org:*` scripts read from here. Changes flow: `org-config.yaml` -> org scripts -> `config/defaults.ts` -> CDK stacks.

## Test Pattern

Tests use the Node.js built-in test runner (`node:test`). Source in `test/`, compiled to `dist/test/`, run from there. Tests cover config validation, CDK stack synthesis assertions, org script logic, and CLI argument parsing.

## Framework Files

These files define the repo's core plumbing -- only modify when explicitly intending to change the framework:

- `config/schema.ts`
- `config/load-config.ts`
- `config/defaults.ts`
- `lib/app-stage-props.ts`
- `lib/pipeline/app-pipeline-stack.ts`

## Package Naming

Directories under `packages/<app>/` should be named after the Lambda/resource's purpose, not the CDK stack name. E.g., `unified-handler/` not `rest-api/`.

## Code Style

- Biome for linting and formatting (replaces ESLint/Prettier)
- Double quotes for strings
- Space indentation
- Husky pre-commit hook runs `lint-staged` -> `biome check --write`
- TypeScript strict mode, target ES2022, CommonJS modules
- Imports use `node:` prefix for Node builtins (e.g., `node:fs`, `node:path`)
- Always use arrow notation for functions (e.g., `const fn = () => {}` not `function fn() {}`)
- Enum members must be CAPITALIZED (e.g., `USE_EXISTING`, not `UseExisting`)
